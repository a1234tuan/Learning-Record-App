# Learning Record App

一个本地优先的学习日志应用，支持 Web/PWA 和 Android APK。它把每天的学习记录、图片、录音、附件、OCR 文本和 AI 问答沉淀成本地知识库，适合 408 备考或长期自学复盘。

应用没有账号系统，核心数据保存在本机 IndexedDB / Android WebView 存储中。完整备份、AI API Key、聊天记录等敏感边界都按本地优先原则处理。

## 当前导航

底部主导航：

- **今天**：新建当天学习记录、查看今日记录、进入收藏。
- **日志**：按月份热力图和有记录日期回看日志，右上角进入全局搜索。
- **分类**：按学科查看记录，并在管理模式中新增、改名、归档和排序学科。
- **录音**：按学科文件夹查看所有日志中的录音文件，支持搜索、重命名和播放。
- **更多**：备份恢复、AI 工具、回收站、统计和应用设置入口。

侧栏/更多入口还保留 AI 问答、统计、设置等快捷入口。

## 主要功能

### 今天页

- 首页只保留新建学习记录的主按钮：`+ 新建 {学科} 记录`。
- 学科新增和管理统一放在“分类/学科管理”中，首页不再出现重复的新增学科入口。
- 收藏夹入口放在首页右上角星标按钮，避免占用底部空间。
- 今日记录卡片支持打开详情、收藏、按当天开启 AI 问答。

### 日志页

- 顶部标题为“日志回看”，右上角提供 `全局搜索` 按钮。
- 搜索不再单独占用底部 tab，点击日志页搜索按钮进入原搜索界面。
- 月度热力图采用紧凑样式，不占用过多首屏空间。
- 日期列表显示“当前月份所有有记录日期”，不是只显示最近 5 天，因此历史日期也能直接进行按天 AI 问答。
- 日期日志条目保留 `AI 问答`，去掉右侧圆圈数字和展开箭头；点击条目主体仍可展开当天学科记录。

### 记录编辑器

- 支持富文本/Markdown 编辑，包括标题、加粗、引用、代码块、任务列表、无序列表和有序列表。
- 有序列表工具栏按钮会插入 Markdown 风格的数字列表；手动输入 `数字 + "." + 空格` 也会触发 ordered-list 规则。
- 支持图片、音频、附件和公式内容。
- 图片可进行 OCR，OCR 文本可进入搜索、导出和 AI 日志上下文。
- 未保存内容会自动保存为草稿；返回、切页、切后台或重启后可以恢复。
- Android 录音使用前台服务：从编辑区开始录音后，息屏、回桌面或切到其他 App 时仍可继续录音；离开编辑页时会自动停止并把已录内容保存进草稿。

### 录音文件夹

- 原“搜索”tab 已改为“录音”tab。
- 录音页按日志学科展示文件夹，例如 OS、计组、计网、数据结构、数学、英语、政治等。
- 文件夹内列出该学科所有日志中引用过的录音文件，显示所属日志标题、录音标题、时长和日期。
- 支持按录音文件名搜索。
- 支持全局重命名录音：在录音页或日志编辑器中改名后，所有引用同步更新。
- 点击录音进入播放器并自动播放。
- 播放器支持播放/暂停、上一首/下一首、快退/快进 10 秒、倍速、单录音循环、文件夹循环和顺序播放。

### 分类页

- 按学科查看历史记录。
- 管理模式中可以新增、重命名、归档、恢复和排序学科。
- 新增学科功能只保留在这里，避免首页和编辑器重复出现入口。

### AI 问答与 AI 工具

- AI 问答以“某一天的全部日志”为上下文，而不是单条记录。
- 日志页日期卡片、记录卡片、今天页、分类页和收藏页都可以按当天开启 AI 问答。
- AI 聊天页支持多会话历史、本机聊天记录、最近对话记忆和会话摘要。
- AI 工具页集中管理：
  - AI 问答与聊天记录入口
  - OpenAI 兼容供应商设置
  - API Key 本机保存
  - 图片问答方式
  - 预设提示词
  - AI 材料导出
- AI 材料导出支持按学科 Markdown、知识库 JSON 和纯文本 TXT。

### 更多页

- “更多”根页已改成轻量入口页，不再直接堆叠大块设置表单。
- 工具区包含：
  - 备份与恢复
  - AI 工具
- 应用区包含：
  - 回收站
  - 统计
  - 设置
- 备份与恢复、自动备份、AI 设置、AI 材料导出都进入二级页面，减少根页滚动压力。

### 备份与恢复

- 完整 zip 备份包含日志、学科、设置、图片、音频、附件、OCR 元数据、草稿和回收站等数据。
- 导入恢复只接受完整备份 zip，并采用覆盖式恢复。
- 导入前会二次确认；导入过程会显示解析、恢复、成功、失败或取消状态。
- 自动备份可绑定文件夹并写入 `study-journal-latest.zip`。
- API Key 和 AI 聊天记录不会进入完整备份。

### 回收站与统计

- 删除记录会先进入回收站，保留 30 天，可恢复或永久删除。
- 统计页展示记录数量、活跃天数、资源数量、学科分布和近期趋势。

## AI 上下文边界

AI 日志问答会发送：

- 日期
- 学科
- 记录标题
- 正文文本
- 公式内容
- 已完成 OCR 的图片文字
- 当前用户输入
- 最近若干轮有效问答记忆

不会发送：

- 未保存草稿
- 回收站记录
- 原始音频文件
- 普通附件原文
- API Key
- 本机 AI 聊天记录本身

图片问答输入区可以临时上传图片，具体处理方式取决于 AI 设置中的“图片问答方式”：本地 OCR 后转文字、直接发送给支持视觉的模型，或关闭图片发送。

## Android 与 Web 差异

- Android 端使用 Capacitor 打包，支持原生能力桥接。
- Android 端录音使用前台服务，录音开始后可以在息屏、回桌面或切换 App 时继续保持。
- Android 端 AI 请求优先走原生 HTTP 桥接，更容易避开 WebView/CORS 限制。
- Web/PWA 端受浏览器权限和后台策略限制，不承诺息屏或后台持续录音。
- Web 端 AI/OCR 可能受到第三方接口 CORS 限制，必要时使用 Android 端或支持 CORS 的中转接口。

## 数据与隐私

主要本地数据包括：

- 日志记录
- 学科设置
- 图片、音频、附件 Blob
- OCR 文本
- 收藏记录
- 回收站记录
- 未保存草稿
- AI 非敏感设置
- 本机 AI 聊天记录

重要提醒：卸载 App、清除应用数据、清理浏览器站点数据，都可能删除本地库。请定期导出完整 zip 备份，或配置自动备份。

## Web 端使用

安装依赖：

```powershell
cd D:\NoteProject
npm.cmd install
```

启动开发服务：

```powershell
npm.cmd run dev
```

浏览器打开终端显示的地址，通常是：

```text
http://localhost:5173/
```

局域网访问：

```powershell
npm.cmd run dev -- --host 0.0.0.0
```

生产构建：

```powershell
npm.cmd run build
npm.cmd run preview
```

## Android APK

构建 debug APK：

```powershell
cd D:\NoteProject
npm.cmd run android:build:debug
```

APK 输出路径：

```text
android\app\build\outputs\apk\debug\app-debug.apk
```

通过 adb 安装：

```powershell
adb install -r android\app\build\outputs\apk\debug\app-debug.apk
```

也可以把 APK 发送到手机后手动安装。首次安装可能需要允许“安装未知来源应用”。

## 技术栈

- React + TypeScript + Vite
- Dexie / IndexedDB
- TipTap 富文本编辑器
- JSZip 备份恢复
- Vite PWA
- Capacitor Android
- OpenAI 兼容 AI 接口
- PaddleOCR 图片文字识别

## 当前限制

- 暂无实时云同步，双端同步依赖完整 zip 备份导入或自动备份文件。
- 导入是覆盖式恢复，不做冲突合并。
- Web 端 AI/OCR 可能受到第三方接口 CORS 限制。
- 当前 APK 是 debug 构建，适合个人安装测试，不是应用商店发布版。
- AI 直接看图能力取决于所选模型和中转接口是否真正支持视觉输入。
- 聊天记录和 API Key 只保存在本机，不进入完整备份。

## 开发注意

- 当前 `appId/applicationId` 保持为 `com.noteproject.study408`，用于兼容已安装版本的数据。
- `dist/`、`android/app/build/`、`*.apk` 不应提交到仓库。
- 不要把真实 API Key、OCR Token 或其他密钥提交到仓库。
- 如果后续接入云同步，建议先实现“云端 latest zip 备份”，再逐步升级为结构化增量同步。
